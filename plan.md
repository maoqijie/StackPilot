# StackPilot CloudPulse 全页面重构

## 目标

依据 `DESIGN.md` 将现有前端重构为统一、响应式的 CloudPulse 运维控制台，同时保留所有页面功能、数据流、权限保护、表单校验和操作反馈。

- [x] 重构桌面、平板和手机共用的应用框架，迁移旧 `#mobile` 路由。
- [x] 重写按钮、表单、状态、表格、卡片、抽屉、弹窗和导航样式。
- [x] 重排工作台、资源管理、运维、安全、权限和设置页面。

- [x] 完成类型检查、lint、测试、构建和多视口浏览器验收。

## 验收记录

- `npm run typecheck`：通过。
- `npm run lint`：通过。
- `npm test`：通过，Contracts、Config、Controller、Web、Agent 与架构测试全部成功。
- `npm run build`：通过；Vite 仅保留原有的大 chunk 提示。
- `npm run test:e2e`：通过，桌面与移动 Chromium 共 4 项。
- 主题测试：浅色默认、无效值回退、深色恢复、切换和 `localStorage` 持久化均通过。
- 路由测试：旧 `#mobile` 自动替换为 `#overview`，其它深链保持兼容。
- Playwright：检查 45 个路由在 1280×800、1024×768、768×1024、430×932、390×844、375×812 下的 270 个组合，未发现页面级横向溢出。
- 视觉截图：覆盖工作台、主机、数据库、终端、设置、防火墙和审计的桌面/手机状态；明暗主题与本地字体加载正常。
- 移动导航：验证遮罩抽屉、背景层、主内容 `inert`、Escape 关闭和焦点恢复。
- Lighthouse：当前 Windows 环境中的临时 Lighthouse CLI 无输出且不生成报告，无法提供可信分数；真实浏览器检查与生产构建已完成。
- 已知环境现象：Windows 上 `/api/overview/current-user-crontab` 返回 500，属于现有平台能力限制，页面会显示对应失败状态，与本次样式重构无关。
