# 前端验收清单

本文档用于验收 StackPilot 前端首版体验。验收目标是形成以 1Panel 为主参考、宝塔和 MCSM 为辅助参考的专业简洁运维控制台，默认浅色主题，暗色主题可选。

## 产品定位

- [ ] 信息架构以服务器运维与应用发布为核心，不做营销页或装饰性首页。
- [ ] 交互密度接近 1Panel：信息清晰、操作直接、状态反馈明确。
- [ ] 宝塔仅作为服务器管理与防火墙操作参考，不复制其复杂导航和视觉噪声。
- [ ] MCSM 仅作为服务实例、运行状态、日志/控制台类场景的辅助参考。
- [ ] 文案保持专业、短句、可执行，不出现教程式大段说明。

## 视觉风格

- [ ] 默认主题为浅色，整体干净、克制、留白适中。
- [ ] 暗色主题作为可选能力，颜色层级与浅色主题一致。
- [ ] 主色用于关键动作、选中态和重要状态，不大面积铺满页面。
- [ ] 页面不使用夸张渐变、大型装饰背景、拟物装饰或营销式 hero。
- [ ] 表格、列表、状态标签、表单和弹窗视觉统一。
- [ ] 图标服务于识别与操作，不替代关键状态文本。
- [ ] 卡片仅用于独立信息块，不嵌套卡片。

## 布局与导航

- [ ] 首屏直接进入控制台工作界面。
- [ ] 左侧导航或等效主导航包含：总览、服务器、服务、防火墙、发布、审计、设置。
- [ ] 当前页面、可用动作、危险动作有明确视觉区分。
- [ ] 页面主体宽度、表格列、工具栏和筛选区在常见桌面尺寸下稳定不跳动。
- [ ] 空状态、加载态、错误态、禁用态完整可见。
- [ ] 危险操作必须有二次确认或等效保护。

## 页面验收

### 总览

- [ ] 展示服务器数量、在线状态、服务运行状态、发布状态和近期审计事件。
- [ ] 关键指标能区分正常、警告、异常。
- [ ] 提供跳转到服务器、服务、发布、审计的明确入口。

### 服务器

- [ ] 支持查看服务器列表、连接状态、系统资源、标签或分组。
- [ ] 支持进入服务器详情。
- [ ] 服务器详情展示 CPU、内存、磁盘、网络、系统信息和最近活动。
- [ ] 离线、连接失败、权限不足等状态有清晰提示。

### 服务

- [ ] 支持查看服务列表、运行状态、所属服务器、端口和最近更新时间。
- [ ] 支持启动、停止、重启等常见操作入口。
- [ ] 支持查看服务详情、环境信息、日志入口和运行健康状态。
- [ ] 运行中、已停止、异常、部署中状态区分明确。

### 防火墙

- [ ] 支持查看规则列表，包含端口、协议、来源、策略、状态和备注。
- [ ] 支持新增、编辑、启用、禁用、删除规则的入口。
- [ ] 高风险规则有醒目标识。
- [ ] 表单校验覆盖端口范围、协议、来源地址和必填项。

### 发布

- [ ] 展示发布项目、目标服务器、版本、状态、操作者和时间。
- [ ] 支持查看发布详情、步骤进度、日志入口和失败原因。
- [ ] 发布中、成功、失败、回滚中状态清晰。
- [ ] 失败场景提供重试、回滚或查看日志入口。

### 审计

- [ ] 展示操作人、操作类型、目标对象、结果、时间和来源信息。
- [ ] 支持按时间、操作人、对象、结果筛选。
- [ ] 失败和危险操作在列表中容易识别。
- [ ] 审计详情能还原关键上下文。

### 设置

- [ ] 包含基础配置、主题偏好、通知偏好和安全相关配置入口。
- [ ] 配置项分组清晰，默认值明确。
- [ ] 保存、取消、重置等动作语义清楚。
- [ ] 保存成功、失败和校验错误均有反馈。

## 响应式验收

- [ ] 桌面端 1440px、1280px 下布局稳定，无文本遮挡或横向溢出。
- [ ] 平板端 1024px、768px 下导航、表格和详情页可用。
- [ ] 移动端 430px、390px、375px 下主流程可访问。
- [ ] 小屏下表格提供卡片化、横向滚动或列优先级策略。
- [ ] 工具栏、筛选区和批量操作在小屏下不挤压主要内容。
- [ ] 弹窗、抽屉、下拉菜单在小屏下不超出视口。
- [ ] 触控目标尺寸不小于 40px。

## 可用性与可访问性

- [ ] 主要操作可通过键盘聚焦和触发。
- [ ] 焦点态清晰可见。
- [ ] 文本、状态标签和按钮满足基础对比度要求。
- [ ] 表单字段有关联标签、错误提示和可恢复路径。
- [ ] 图标按钮具备可访问名称或 tooltip。
- [ ] 页面标题、导航结构和主要区域语义清楚。

## 性能与质量门槛

- [ ] 构建无 TypeScript、lint 或运行时错误。
- [ ] 首屏无明显布局抖动。
- [ ] 关键页面无未处理异常、控制台错误或资源 404。
- [ ] Lighthouse Performance 分数必须大于等于 98。
- [ ] Lighthouse 验收至少覆盖默认入口页；如存在登录态限制，需记录真实浏览器替代验证结果。
- [ ] 桌面与移动视口均需进行真实浏览器截图或等效页面级检查。
