# Upgrade to 0.3.0-preview.5

Status: preview. This release connects the systemd service and log workbench to bounded, read-only Agent snapshots. It does not change Controller schema 7 or the Agent protocol version.

Direct upgrade sources: StackPilot 0.3 preview releases and Controller SQLite schemas 1 through 7. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.5.sqlite3
```

Deploy Controller and Web first, then deploy the Agent on each monitored Linux node. The Agent systemd unit now joins the `systemd-journal` supplementary group so it can read system journal entries without root. Run the installer or synchronize the packaged unit, execute `systemctl daemon-reload`, and restart `stackpilot-agent.service` explicitly during the planned rollout. Verify the service still runs as `stackpilot-agent`, has no ambient capabilities, and can read only the journal access granted by the operating system group.

Older Agents remain compatible but report no systemd snapshot; the Web labels those nodes as awaiting collection instead of displaying fixture data. Updated Agents negotiate `systemd-snapshot-v1` before adding the optional heartbeat field, so the Controller-first rollout order is required. Service collection is bounded to 64 units and four recent journal entries per unit. Common authorization, token, password, secret and API-key forms are redacted before upload, but operators must still avoid logging secrets and should review journal access according to local policy.

Controller schema 7 has no automatic down migration. To roll back, stop the new services, verify the pre-upgrade backup checksum, switch the release symlink to the previous version, restore the verified SQLite backup only when the previous Controller requires it, restore the previous systemd units, run `systemctl daemon-reload`, and start the previous Controller and Agent.
