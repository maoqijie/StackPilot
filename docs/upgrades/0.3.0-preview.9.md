# Upgrade to 0.3.0-preview.9

Status: preview. This release connects the deployment workbench to the existing site-management control plane and repairs the disposable systemd installer verification fixture without weakening production release integrity checks. It adds a read-only deployment projection from saved plans, operations and releases, removes browser-only deployment mutations, and preserves the existing reauthentication, idempotency and Agent capability boundaries for real Git deployments.

Direct upgrade sources: Controller SQLite schemas 1 through 7 and StackPilot 0.3 preview releases. This release does not change schema 7. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.9.sqlite3
```

Deploy Controller and Web together so the shared deployment contract and UI remain version-aligned. Then deploy the site helper and Agent from the same revision. Verify authenticated `GET /api/deployments`, the `#deploy-prod` desktop and mobile views, and at least one complete 10-second polling interval after cutover.

The deployment page intentionally does not expose a rollback action because the current Agent/helper protocol has no bounded historical-release rollback task. Failed activations still retain the helper's existing automatic restoration behavior. Do not replace this boundary with arbitrary shell execution.

To roll back, stop the new Controller, verify the `preview.7` pre-upgrade backup checksum, switch the release symlinks to the previous releases, restore that verified SQLite backup if required, restore the prior systemd units, run `systemctl daemon-reload`, and restart the previous Controller, Agent and helper.
