# Upgrade to 0.3.0-preview.33

Status: preview. This release hardens the real UFW rule-management backend while preserving the audit pagination and SQLite schema 9 introduced in `0.3.0-preview.32`. The Agent protocol remains `1.1`.

Direct upgrade sources are Controller SQLite schemas 1 through 9 and earlier StackPilot 0.3 preview releases. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x and UFW installed. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.33.sqlite3
```

Deploy Controller/Web and cert-helper together, synchronize sysusers, tmpfiles and both firewall-helper systemd units, then enable `stackpilot-firewall-helper.socket` before restarting Controller. The tmpfiles setup must create both `/run/ufw.lock` and `/run/xtables.lock`. Do not enable or disable UFW and do not change its default policy. Confirm UFW retains its pre-upgrade state and that an inactive UFW keeps mutations disabled.

Confirm unauthenticated `GET /api/firewall/rules`, `GET /api/firewall/open-ports` and `GET /api/firewall/deny-records` return `401`. With an authorized session, confirm all three endpoints return backend-provided collection timestamps and no fixture rows. Run `npm run audit:verify --workspace @stackpilot/controller`, then observe a complete 10-second polling cycle at desktop, mobile and intermediate widths without overlap, horizontal overflow or console errors.

Schema 9 remains unchanged from `0.3.0-preview.32`. To roll back to that release, disable the firewall helper socket, switch Controller/Web and cert-helper release symlinks and unit files, and restart Controller. Restoring the database is not required unless a schema 8 or earlier release is selected.
