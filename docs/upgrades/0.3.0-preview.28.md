# Upgrade to 0.3.0-preview.28

Status: preview. This release replaces the firewall-rule fixtures with real UFW collection and capability-scoped local management. It preserves the real listening-port, deny-record, audit and schedule-execution backends from earlier releases and does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.28.sqlite3
```

Deploy Controller/Web and the cert-helper package, synchronize sysusers, tmpfiles and the new `stackpilot-firewall-helper.socket` and `stackpilot-firewall-helper@.service` units, then enable the firewall helper socket before restarting Controller. Do not enable UFW or change its default policy during this upgrade. When UFW is inactive, the rule page remains read-only and reports the backend state.

Confirm unauthenticated `GET /api/firewall/rules` returns `401`. With an authorized session, confirm it returns `engine: "ufw"`, the actual hostname, backend collection time, `active: false` when UFW is inactive, and no fixture rows. Observe at least one complete 10-second polling interval on desktop and mobile without overlap, layout shifts or console errors.

Schema 8 remains unchanged. To roll back, disable the firewall helper socket, switch Controller/Web and cert-helper release symlinks and unit files to `0.3.0-preview.27`, and restart Controller. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
