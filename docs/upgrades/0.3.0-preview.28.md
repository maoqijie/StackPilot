# Upgrade to 0.3.0-preview.28

Status: preview. This release preserves the real audit, schedule execution and firewall deny backends through `0.3.0-preview.27` and corrects the Agent's interpretation of an empty Linux firewall journal query. It does not change Controller schema 8 or the Agent protocol version.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.28.sqlite3
```

Deploy Controller and Web first, then deploy Agent on each monitored Linux node. Confirm the Agent process retains read-only `systemd-journal` group membership. When the fixed journal query has no matches, authenticated `GET /api/firewall/deny-records` must return `collectionStatus: complete`, an empty `records` array and a real backend `collectedAt`; permission failures must remain `unavailable`. Observe at least one complete 10-second Web polling interval at desktop and mobile widths.

Schema 8 remains unchanged. To roll back, switch Controller, Web and Agent to the previous compatible release and restart the services. Database restoration is not required for this release.
