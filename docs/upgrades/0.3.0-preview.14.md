# Upgrade to 0.3.0-preview.14

Status: preview. This release strengthens credential redaction for systemd journal snapshots and locks the read-only Web monitoring behavior to the standard visibility-aware 10-second polling contract. It does not add systemd lifecycle operations or change the Controller database schema.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.14.sqlite3
```

Deploy Controller and Web first, then deploy the Agent so protocol feature negotiation remains backward compatible. Verify authenticated `GET /api/systemd/services` returns `Cache-Control: no-store`, real node-scoped services and backend collection timestamps. Observe at least one complete 10-second Web polling interval, open a journal drawer at desktop and mobile widths, and confirm the selected service remains stable across refreshes. No systemd mutation endpoint should be present.

Schema 8 remains unchanged. To roll back, switch the Controller/Web and Agent release symlinks to the previous compatible release; database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
