# Upgrade to 0.2.0-preview.1

Status: preview. This release adds production deployment definitions, trusted proxy handling, schema 2 release metadata, HTTPS E2E gates, SBOM/checksum generation and a keyless signing workflow.

Direct upgrade source: `0.1.x` database schema 1. Before changing binaries, verify Node 22.x, at least 512 MiB free space, HTTPS origins, TLS files and Agent versions, then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.2.0-preview.1.sqlite3
```

The upgrade command runs preflight, creates an online backup and SHA-256 sidecar, applies transactional migrations, and verifies the audit chain. Any failure stops publication. Do not start the new Controller if migration failed.

Rollback: stop Controller, verify the pre-upgrade checksum, switch `current` back to the prior application, restore the schema-1 backup, and start the prior Controller. Schema 2 has no down migration; never point the prior binary at schema 2. Preserve the failed schema-2 database for diagnosis.

Known issues: the minified Web entry chunk remains about 650 kB; Firefox is not yet an automated E2E gate; Docker/systemd runtime gates require Linux CI; there is no stable support SLA or published predecessor release artifact.
