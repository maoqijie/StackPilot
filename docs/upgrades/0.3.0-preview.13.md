# Upgrade to 0.3.0-preview.13

Status: preview. This release retains the schema 8 deployment and site-rollback workbenches from `0.3.0-preview.12`, preserves the idempotent site-plan node-scope hardening, and fixes a same-process site-helper lock handoff race found by the Linux CI concurrency regression.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. The latest schema remains schema 8; no additional migration is introduced in this release. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.13.sqlite3
```

Deploy Controller, Web and cert-helper together. After cutover, verify `/healthz`, `/readyz`, authenticated `GET /api/deployments`, `#deploy-staging` at desktop and mobile widths, and at least one complete 10-second polling interval. Keep the systemd cert-helper unit's outer `flock` boundary enabled, and verify concurrent activation requests remain serialized without `ENOENT` or `EINVAL` lock-owner failures.

Schema 8 has no automatic down migration. To roll back StackPilot itself, stop the new Controller, verify the checksum of the backup created immediately before this upgrade, switch each affected release symlink to its previous release, restore that verified backup only when the schema changed during the attempted rollout, and restart the previous Controller, Web and helper versions together.
