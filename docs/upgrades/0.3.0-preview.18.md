# Upgrade to 0.3.0-preview.18

Status: preview. This release replaces the Web-only firewall fixtures with real UFW rule collection and a capability-scoped local management boundary. It does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.18.sqlite3
```

Deploy Controller/Web and the cert-helper package, synchronize the new `stackpilot-firewall-helper.socket` and `stackpilot-firewall-helper@.service` units, run `systemd-sysusers` and `systemd-tmpfiles`, then enable the firewall helper socket before restarting the Controller. Do not enable UFW or change its default policy as part of this application upgrade. On hosts where UFW is inactive, the page remains read-only and reports that state explicitly.

Confirm authenticated `GET /api/firewall/rules` returns the actual UFW status, hostname and backend collection timestamp. Confirm unauthenticated access returns `401`, users without `firewall:read` receive `403`, and mutations require CSRF, `firewall:operate`, recent reauthentication and a StackPilot-managed rule identity. Observe one complete 10-second Web polling interval without layout changes or duplicate requests.

Schema 8 remains unchanged. To roll back, disable the firewall helper socket, switch Controller/Web and cert-helper release symlinks to the previous compatible release, restore the previous systemd unit files and restart the Controller. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
