# Upgrade to 0.3.0-preview.18

Status: preview. This release connects the managed schedule workflow to the Controller service account's real user crontab under the hardened systemd unit. It does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.18.sqlite3
```

Deploy Controller and Web together. Enable schedule mutations only when intended by setting `STACKPILOT_ENABLE_CRONTAB_WRITE=1` in the protected Controller environment. Confirm authenticated `GET /api/overview/current-user-crontab` returns the Controller service account's managed tasks, then use a reversible disabled-task create/delete check to verify the managed block without executing a command. Observe at least one complete 10-second Web polling interval at desktop and mobile widths.

Schema 8 remains unchanged. To roll back, reset `STACKPILOT_ENABLE_CRONTAB_WRITE=0`, switch Controller and Web to the previous compatible release, restore the previous systemd unit, and restart Controller. Database restoration is not required for this release.
