# Upgrade to 0.3.0-preview.30

Status: preview. This release preserves the Agent-backed firewall deny records and schedule execution/calendar fixes through `0.3.0-preview.29`, then adds real Controller-host UFW rule management plus listening-port collection. It does not change Controller schema 8 or the Agent protocol version.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x and UFW installed on the Controller host. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.30.sqlite3
```

Deploy the cert-helper release and synchronize its systemd units before restarting the Controller. Confirm `stackpilot-firewall-helper.socket` is active, owned by `root:stackpilot-firewall`, and unavailable to the Agent account. UFW remains in its existing active or inactive state; StackPilot never enables or disables it. When UFW is inactive, authenticated reads still report the backend state and mutations remain locked.

Deploy Controller and Web together, then verify authenticated `GET /api/firewall`, `GET /api/firewall/open-ports` and `GET /api/firewall/deny-records` return backend timestamps and no fixture rows. Observe at least one complete 10-second polling interval at desktop and mobile widths. To roll back, switch Controller, Web and cert-helper to the previous compatible release and restore the previous units. Database restoration is not required for this release.
