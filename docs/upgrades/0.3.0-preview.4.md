# Upgrade to 0.3.0-preview.4

Status: preview. This release replaces the systemd workbench fixtures with authenticated Controller APIs backed by the existing root-only local helper socket. It adds real unit inventory, bounded journal reads and explicitly allowlisted start, stop and restart actions. Database schema remains at version 7 and Agent protocol compatibility is unchanged.

Direct upgrade sources: Controller SQLite schemas 1 through 7 and StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material, the master key and `/etc/stackpilot-site-helper/helper.env`.

Configure `STACKPILOT_SYSTEMD_MANAGED_UNITS` in `/etc/stackpilot-site-helper/helper.env` with the exact units operators may start, stop or restart. Inventory and journal reads remain available for valid systemd units, but mutation buttons are omitted for units outside this allowlist. Do not include `stackpilot-controller.service`, because a synchronous Controller request cannot safely restart its own process.

Deploy in this order: certificate/site helper, Controller and Web, database helpers, then Agent. The helper must understand the new fixed systemd protocol before the Controller exposes the corresponding API. Synchronize the helper systemd unit, restart the helper socket if required, then cut over the matching Controller and Web release.

After deployment, sign in and open `#systemd-active`. Verify that the displayed hostname, unit count and collection timestamp match the target host, wait through one complete 10-second polling interval, open one journal drawer, and test one non-critical allowlisted unit action with current-password reauthentication. Confirm that units outside the allowlist have no mutation controls.

To roll back, switch the helper, Controller and Web release symlinks to `0.3.0-preview.3`, synchronize their systemd units and restart those components. No database restore is required because this release does not change schema 7. Keep the pre-upgrade backup until the rollback window closes.
