# Upgrade to 0.3.0-preview.1

Status: preview. This release replaces the database instance, backup and slow-query fixtures with real PostgreSQL, MySQL and MariaDB monitoring and operations. It introduces Controller schema 6, Agent protocol 1.1 and the root-only `database-helper` boundary.

Direct upgrade sources: Controller SQLite schemas 1 through 5. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, preserve Controller configuration, TLS material and the master key, then run the release preflight and upgrade commands from the new release:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.1.sqlite3
```

Deploy in this order: Controller and Web, `database-helper`, then Agent. Controller 0.3 accepts protocol 1.0 and 1.1 Agents, but database capabilities require protocol 1.1 and a reachable helper. Do not connect a protocol 1.1 Agent to an older Controller. Existing database instances must be registered locally on their node so credentials never pass through the browser or Controller.

Schema 6 has no automatic down migration. To roll back, stop the new Controller, verify the pre-upgrade backup checksum, switch the release symlink to the previous version, restore the verified SQLite backup, and start the previous Controller. Stop the new Agent and helper before restarting the old Controller. Database data directories remain independent of the Controller rollback.

Production limits: do not enable a full local backup for the existing large PostgreSQL cluster unless capacity preflight proves sufficient space. Do not restart that cluster merely to enable historical slow-query extensions. In-place restore is limited to StackPilot-created managed instances. Validate install, backup, read-only mode, session termination, index creation and restore only on isolated high-port test instances after proving the external network boundary blocks those ports.

Compatibility note: package drivers target Debian 12, Ubuntu 24.04, Rocky/Alma 9, Fedora 42, Alpine 3.22 and Arch, but this preview does not claim those Agent/helper targets as certified until pinned per-OS installation, multi-instance, backup and restore integration jobs pass.
