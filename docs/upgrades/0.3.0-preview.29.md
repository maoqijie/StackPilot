# Upgrade to 0.3.0-preview.29

Status: preview. This release adds real next-run timestamps and ordering to the Controller-backed schedule calendar while retaining the command-versioned automatic and manual execution records from `0.3.0-preview.28`. It does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x and the distribution `cron` package. Before changing the running release, verify `/usr/bin/crontab`, the `crontab` group and `/var/spool/cron/crontabs`, create an online SQLite backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.29.sqlite3
```

Deploy Controller and Web together, synchronize Controller sysusers and the base systemd unit, then restart during a controlled maintenance window. Confirm authenticated `GET /api/overview/current-user-crontab` returns `200`, `Cache-Control: no-store`, an ISO `scannedAt`, a nullable ISO `nextRunAt` for every task and the existing `lastExecution` record when available. Verify `#schedule-calendar` orders enabled jobs by the Controller host's cron time zone, keeps unavailable or disabled times explicit, preserves execution details, and remains stable through one complete 10-second polling interval at desktop and mobile widths.

Schema 8 remains unchanged. To roll back, set `STACKPILOT_ENABLE_CRONTAB_WRITE=0` if the schedule path is implicated, switch the Controller/Web release symlink and base unit to the previous compatible release, and restart Controller. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
