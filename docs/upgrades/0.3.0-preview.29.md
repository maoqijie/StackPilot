# Upgrade to 0.3.0-preview.29

Status: preview. This release connects persistent audit-export APIs, adds Controller schema 9 for export metadata, and adds real next-run timestamps and ordering to the Controller-backed schedule calendar. It preserves the real audit, command-versioned schedule execution and firewall deny backends through `0.3.0-preview.28` without changing the Agent protocol version.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x and the distribution `cron` package. Before changing the running release, verify the current SQLite database, `/usr/bin/crontab`, the `crontab` group and `/var/spool/cron/crontabs`, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.29.sqlite3
```

Deploy Controller and Web together, synchronize Controller sysusers and the base systemd unit, then restart during a controlled maintenance window. Confirm authenticated audit-export creation and listing, CSV and JSON streaming downloads, SHA-256 integrity, and at least one complete 10-second polling interval on `#audit-export`. Confirm `GET /api/overview/current-user-crontab` returns `200`, `Cache-Control: no-store`, an ISO `scannedAt`, a nullable ISO `nextRunAt` for every task and the existing `lastExecution` record when available. Verify `#schedule-calendar` orders enabled jobs by the Controller host's cron time zone and remains stable through one complete polling interval. Also confirm existing global and failed audit views still load from `GET /api/audit`, and the firewall deny workbench preserves the `0.3.0-preview.28` empty-query behavior.

Schema 9 adds only `audit_exports`. To roll back across schema 9, stop the Controller, set `STACKPILOT_ENABLE_CRONTAB_WRITE=0` if the schedule path is implicated, switch the Controller and Web release symlinks and base unit, restore the verified pre-upgrade schema 8 backup, and restart the previous release.
