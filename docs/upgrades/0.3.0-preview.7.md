# Upgrade to 0.3.0-preview.7

Status: preview. This release keeps the multi-node read-only systemd backend, Controller schema 7 and Agent protocol from `0.3.0-preview.6`. It hardens journal redaction and legacy service-name validation, distinguishes transitional systemd states, and bounds large aggregated workbench rendering.

Direct upgrade sources: StackPilot 0.3 preview releases and Controller SQLite schemas 1 through 7. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.7.sqlite3
```

Deploy Controller and Web first, then deploy the Agent on each monitored Linux node. Confirm the Agent remains non-root with the `systemd-journal` supplementary group, sends negotiated `systemd-snapshot-v1` heartbeats, and exposes no plaintext JSON, query-string or connection-string credentials in the Controller response. Verify the systemd workbench after at least one heartbeat and one complete 10-second Web polling interval at desktop and mobile widths.

Controller schema 7 has no automatic down migration. To roll back, stop the new services, switch Controller, Web and Agent release symlinks to `0.3.0-preview.6`, restore the previous systemd units, run `systemctl daemon-reload`, and start the previous Controller and Agent. A database restore is not required because this release does not change the schema.
