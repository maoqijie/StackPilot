# Upgrade to 0.3.0-preview.12

Status: preview. This release retains the schema 8 deployment and site-rollback workbenches from `0.3.0-preview.11` and hardens idempotent site-plan reads against later node-scope reductions.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. The latest schema remains schema 8; no additional migration is introduced in this release. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.12.sqlite3
```

Deploy Controller and Web together. After cutover, verify `/healthz`, `/readyz`, authenticated `GET /api/deployments`, `#deploy-staging` at desktop and mobile widths, and at least one complete 10-second polling interval. A principal whose node scope excludes a previously created plan must receive `403` when the same idempotency key is replayed, even when the new request names a currently authorized node.

Schema 8 has no automatic down migration. To roll back StackPilot itself, stop the new Controller, verify the checksum of the backup created immediately before this upgrade, switch the release symlink to the previous release, restore that verified backup only when the schema changed during the attempted rollout, and restart the previous Controller and Web together.
