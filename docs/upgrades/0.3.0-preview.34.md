# Upgrade to 0.3.0-preview.34

Status: preview. This release connects persistent audit-export APIs and adds Controller schema 10 for export metadata. It preserves the server-side audit filtering, cursor pagination and hardened UFW backend through `0.3.0-preview.33`; the Agent protocol remains `1.1`.

Direct upgrade sources are Controller SQLite schemas 1 through 9 and earlier StackPilot 0.3 preview releases. Schema 10 adds only `audit_exports`; the preceding schema 9 ownership backfill and audit indexes remain unchanged. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x and UFW installed. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.34.sqlite3
```

Deploy Controller/Web and cert-helper together, synchronize sysusers, tmpfiles and both firewall-helper systemd units, then enable `stackpilot-firewall-helper.socket` before restarting Controller. Confirm authenticated audit-export creation and listing, CSV and JSON streaming downloads, SHA-256 integrity, and at least one complete 10-second polling interval on `#audit-export` at desktop and mobile widths. Also confirm `GET /api/audit` retains server-side filters and stable cursor pagination, and run `npm run audit:verify --workspace @stackpilot/controller`.

To roll back to `0.3.0-preview.33`, stop the new Controller, restore the verified pre-upgrade schema 9 database backup, switch Controller, Web and cert-helper to the previous release, synchronize its systemd units, and restart. Do not switch only the release symlink while leaving a schema 10 database in place.
