# Upgrade to 0.3.0-preview.23

Status: preview. This release preserves the schedule idempotency and native cron deployment hardening from `0.3.0-preview.22` and completes the real Controller audit integration across global, failed, database and export views. It does not change Controller schema 8, the Agent protocol or helper request contracts.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.23.sqlite3
```

Deploy Controller and Web together. Confirm authenticated `GET /api/audit?limit=200` returns persisted events and `collectedAt`, `GET /api/audit?result=failed` returns only recognized failure outcomes, and `GET /api/audit?actionPrefix=database.` applies the prefix before limiting. Users without the explicitly global `audit:read` permission must not access the route, navigation or direct page rendering. Inspect a real event detail, download the current filtered CSV, and observe a complete 10-second polling interval while filters and the selected stable event remain intact.

Schema 8 remains unchanged. To roll back, switch the Controller/Web release symlink to `0.3.0-preview.22` and restart the Controller. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
