# Upgrade to 0.3.0-preview.23

Status: preview. This release retains the real Controller firewall listener, audit and schedule backends and connects the firewall deny workbench to normalized Linux kernel firewall events collected by StackPilot Agent. It does not change the Controller database schema or Agent protocol version.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.23.sqlite3
```

Deploy Controller and Web first, then deploy Agent on each monitored Linux node. The existing systemd unit grants the Agent read-only `systemd-journal` group membership. Confirm the Agent still runs as `stackpilot-agent` without ambient capabilities, then verify authenticated `GET /api/firewall/deny-records` returns `200`, an actual backend `collectedAt` timestamp and records restricted to the principal node scope. Observe at least one complete 10-second Web polling interval at desktop and mobile widths.

Schema 8 remains unchanged. Older Agents remain compatible but do not provide the new snapshot, so the page reports awaiting or unavailable collection until an upgraded Agent completes its first firewall scan. To roll back, switch Controller, Web and Agent to the previous compatible release and restart the services. Database restoration is not required for this release.
