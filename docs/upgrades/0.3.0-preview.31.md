# Upgrade to 0.3.0-preview.31

Status: preview. This release connects persistent audit-export APIs and adds Controller schema 9 for export metadata. It preserves the real UFW rule, listening-port, deny-record, audit and schedule backends through `0.3.0-preview.30` without changing the Agent protocol version.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x and the distribution `cron` package. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.31.sqlite3
```

Deploy Controller and Web together and keep the `0.3.0-preview.30` firewall helper units active. Confirm authenticated audit-export creation and listing, CSV and JSON streaming downloads, SHA-256 integrity, and at least one complete 10-second polling interval on `#audit-export` at desktop and mobile widths. Also confirm existing audit views load from `GET /api/audit`, UFW rules load from `GET /api/firewall/rules`, and schedule responses retain real `nextRunAt` values.

Schema 9 adds only `audit_exports`. To roll back across schema 9, stop the Controller, switch the Controller and Web release symlinks and unit files to `0.3.0-preview.30`, restore the verified pre-upgrade schema 8 backup, and restart the previous release. Keep the firewall helper disabled only if the rollback investigation implicates that boundary.
