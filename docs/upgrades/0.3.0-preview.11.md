# Upgrade to 0.3.0-preview.11

Status: preview. This release adds real site rollback execution through the existing Controller, RemoteTask, Agent and root-only site-helper boundary. It preserves the explicit production and staging deployment environments introduced in preview.10 and keeps all monitoring reads on the standard visibility-aware 10-second interval.

Direct upgrade sources are Controller SQLite schemas 1 through 7 and earlier StackPilot 0.3 preview releases. The latest schema remains schema 8; no additional migration is introduced in this release. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.11.sqlite3
```

Deploy Controller, Web and Agent together. Verify authenticated `GET /api/site-rollbacks` returns node-scoped data with `Cache-Control: no-store`, confirm `#deploy-rollbacks` shows backend collection time, and observe one complete 10-second polling interval. Use a non-protected test site to verify reauthentication and the rollback confirmation boundary before enabling production use.

Schema 8 has no automatic down migration. To roll back StackPilot itself, stop the new Controller, verify the pre-upgrade backup checksum, switch the release symlink to the previous release, restore the verified SQLite backup only when the schema changed during the attempted rollout, and restart the previous Controller, Web and Agent together.
