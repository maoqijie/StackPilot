# Upgrade to 0.3.0-preview.32

Status: preview. This release extends the real audit workbench with server-side filtering, stable cursor pagination and complete raw JSON export, and introduces SQLite schema 9. It preserves the schedule and UFW backends through `0.3.0-preview.31`; the Agent protocol remains `1.1`.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Schema 9 adds the derived `audit_events.node_id` ownership column, backfills existing audit rows without changing hash-chain fields, and creates node, actor, source and outcome indexes. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.32.sqlite3
```

Deploy Controller and Web together. Confirm `GET /api/audit` is `no-store`, filters are applied before the result limit, sequence-cursor pages do not overlap, and a node-scoped query uses `audit_events_node_sequence_idx`. Run `npm run audit:verify --workspace @stackpilot/controller`, observe one complete 10-second polling interval, and inspect the desktop and mobile audit detail drawer.

To roll back to a Controller that supports only schema 8, stop the new Controller, restore the verified pre-upgrade database backup, switch Controller and Web to the previous release, and restart. Do not switch only the release symlink while leaving a schema 9 database in place.
