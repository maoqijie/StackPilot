# Upgrade to 0.3.0-preview.10

Status: preview. This release hardens the real deployment workbench with explicit production and staging plans. Staging plans still perform the real build and preflight workflow, but the Controller rejects activation before any state write or Agent dispatch. Deployment and operation polling are read-only and run at the standard 10-second interval.

Direct upgrade sources are Controller SQLite schemas 1 through 7 and earlier StackPilot 0.3 preview releases. This release introduces schema 8, which adds `site_plans.deployment_environment`, preserves legacy staging refs during migration, updates stored plan payloads, and creates the deployment-history query index. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.10.sqlite3
```

Deploy Controller and Web together so schema, contracts and UI remain aligned. Verify authenticated `GET /api/deployments` returns `Cache-Control: no-store`, confirm `#deploy` defaults to production, inspect `#deploy-staging`, and observe one complete 10-second polling interval. A staging plan must finish preparation without exposing or accepting activation.

Schema 8 has no automatic down migration. To roll back, stop the new Controller, verify the pre-upgrade backup checksum, switch the release symlink to the previous release, restore the verified SQLite backup, and restart the previous Controller and Web together.
