# Upgrade to 0.3.0-preview.21

Status: preview. This release hardens the real Controller crontab integration from `0.3.0-preview.19` with explicit native cron dependencies and user-scoped idempotency for schedule creation and immediate execution. It does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Debian 12 or Ubuntu 24.04 LTS with Node.js 22.x and the distribution `cron` package. Before changing the running release, verify `/usr/bin/crontab`, the `crontab` group and `/var/spool/cron/crontabs`, create an online SQLite backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.21.sqlite3
```

Deploy Controller and Web together, synchronize Controller sysusers and the base systemd unit, then restart during a controlled maintenance window. Confirm authenticated `GET /api/overview/current-user-crontab` returns `200`, an ISO `scannedAt`, and `writeEnabled: true`. Verify API tokens cannot mutate schedules. With a user session, CSRF token, `schedules:write`, and a fresh reauthentication proof for each request, create a disposable disabled task, retry the same idempotency key and confirm no duplicate appears, edit and enable it, immediately execute a harmless command twice with one idempotency key and confirm one execution, then delete the task and confirm the managed crontab block has no residue.

Schema 8 remains unchanged. To roll back, set `STACKPILOT_ENABLE_CRONTAB_WRITE=0` if the schedule path is implicated, switch the Controller/Web release symlink and base unit to the previous compatible release, and restart Controller. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
