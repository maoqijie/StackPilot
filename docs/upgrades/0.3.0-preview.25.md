# Upgrade to 0.3.0-preview.25

Status: preview. This release preserves the real firewall-deny collection introduced in `0.3.0-preview.24`, connects persistent audit-export APIs, and adds Controller schema 9 for export metadata. It does not change the Agent protocol version. Deploy an Agent from `0.3.0-preview.24` or later when upgrading from `0.3.0-preview.23` or earlier and firewall-deny collection is required.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.25.sqlite3
```

Deploy Controller and Web together. Confirm authenticated audit-export creation and listing, CSV and JSON downloads, SHA-256 integrity, and at least one complete 10-second polling interval on `#audit-export`. Also confirm existing global and failed audit views still load from `GET /api/audit`.

Schema 9 adds only `audit_exports`. To roll back across schema 9, stop the Controller, switch the Controller and Web release symlinks, restore the verified pre-upgrade schema 8 backup, and restart the previous release.
