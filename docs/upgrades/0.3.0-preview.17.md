# Upgrade to 0.3.0-preview.17

Status: preview. This release merges Controller-host systemd helper support with Agent node snapshot monitoring. It retains the deployment, rollback and schedule workflows from `0.3.0-preview.16` and does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.17.sqlite3
```

Deploy Controller and Web first, then restart the cert-helper and Agent after switching their versioned release links. Confirm authenticated `GET /api/systemd/services` returns node-scoped snapshots and backend collection timestamps, local helper journal reads remain bounded, and lifecycle operations still require CSRF, the required permission, recent reauthentication and an allowlisted unit. Observe at least one complete 10-second Web polling interval and confirm the selected service remains stable across refreshes.

Schema 8 remains unchanged. To roll back, switch Controller/Web, cert-helper and Agent release symlinks to the previous compatible release and restart the affected services. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
