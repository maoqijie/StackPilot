# Upgrade to 0.3.0-preview.19

Status: preview. This release connects the failed-schedule workbench to real cron execution results. It does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.19.sqlite3
```

The schedule page remains read-only unless `STACKPILOT_ENABLE_CRONTAB_WRITE=1` is set. When enabled, existing StackPilot-managed cron rows are rewritten to the fixed release runner after their next create, edit or state change. Confirm authenticated `GET /api/overview/current-user-crontab` returns `writable`, backend collection time and typed execution details; verify a controlled failing task appears under `#schedule-failed` after cron runs, then remove the test task.

Schema 8 remains unchanged. To roll back, remove or disable any managed schedules created by this release, switch Controller and Web to the previous compatible release, and restart the Controller. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
