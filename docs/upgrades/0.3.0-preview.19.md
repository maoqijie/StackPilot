# Upgrade to 0.3.0-preview.19

Status: preview. This release connects the firewall open-port workbench to a real Controller host listening-socket backend. It does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.19.sqlite3
```

Deploy Controller and Web together. Confirm authenticated `GET /api/firewall/open-ports` returns `backend: "ss"`, real listeners and a backend collection timestamp. Open `#firewall-open`, inspect desktop and mobile layouts, then observe at least one complete 10-second polling interval. The page intentionally does not mutate UFW or nftables rules; a listening socket does not prove that upstream firewalls permit traffic.

Schema 8 remains unchanged. To roll back, switch the Controller/Web release symlink to the previous compatible release and restart the Controller. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
