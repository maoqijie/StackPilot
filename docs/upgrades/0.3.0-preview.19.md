# Upgrade to 0.3.0-preview.19

Status: preview. This release retains the real managed schedule backend from `0.3.0-preview.18` and connects the failed-audit and global audit views to the authenticated Controller audit repository. It adds backend collection timestamps to the shared response contract and removes the Web audit fixture fallback without changing Controller schema 8, the Agent protocol, or helper request contracts.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.19.sqlite3
```

Deploy Controller and Web together. Confirm authenticated `GET /api/audit?result=failed` returns persisted failure events and `collectedAt`, users without the explicitly global `audit:read` permission cannot access the route or navigation entry, and `#audit-failed` contains only failure outcomes from the real response. Node scope continues to govern node-specific resources; it does not narrow the append-only global security audit. Observe at least one complete 10-second Web polling interval and confirm active filters and event details remain stable while the selected event exists.

Schema 8 remains unchanged. To roll back, switch Controller and Web release symlinks to the previous compatible release and restart both services. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
