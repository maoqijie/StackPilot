# Upgrade to 0.3.0-preview.35

Status: preview. This release keeps the real audit detail drawer fully inside desktop and mobile dynamic viewports. It preserves the persistent audit exports and Controller schema 10 from `0.3.0-preview.34`; the Agent protocol remains `1.1`.

Direct upgrade sources are Controller SQLite schemas 1 through 10 and earlier StackPilot 0.3 preview releases. This release has no database migration. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.35.sqlite3
```

Deploy Controller and Web together. Confirm `#audit-all` renders real Controller events, server-side filters and stable cursor pagination; open an audit detail at desktop and mobile widths and verify every edge remains inside the viewport. Complete at least one 10-second polling interval, verify persistent CSV and JSON downloads against their SHA-256 values, and run `npm run audit:verify --workspace @stackpilot/controller`.

To roll back to `0.3.0-preview.34`, stop the new Controller, restore the verified pre-upgrade schema 10 backup, switch Controller and Web to the previous release, and restart. A database restore is not normally required because this release does not change schema, but keep the verified backup available throughout the cutover.
