# Upgrade to 0.3.0-preview.20

Status: preview. This release corrects IPv4 loopback classification in the real firewall open-port workbench and connects the audit workbench to the real Controller audit repository without simulated audit/export records. It does not change the Controller database schema or Agent protocol.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.20.sqlite3
```

Deploy Controller and Web together. Confirm authenticated `GET /api/audit?limit=200` returns strict events plus `collectedAt`, and `GET /api/audit?limit=200&actionPrefix=database.` filters before limiting. Open `#audit`, inspect a detail record and CSV download on desktop and mobile, then observe at least one complete 10-second polling interval. Users without `audit:read` must not see or directly render the audit workbench.

Also confirm authenticated `GET /api/firewall/open-ports` classifies every `127.x.x.x` listener as `exposure: "loopback"`, then observe at least one complete 10-second polling interval on `#firewall-open`.

Schema 8 remains unchanged. To roll back, switch the Controller/Web release symlink to the previous compatible release and restart the Controller. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
