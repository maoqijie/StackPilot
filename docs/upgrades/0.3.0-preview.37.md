# Upgrade to 0.3.0-preview.37

Status: preview. This release preserves the ACL role, proxy settings and security settings refinements from `0.3.0-preview.36`, and restores the complete release metadata required by strict dependency installation and the signed release workflow. Controller schema remains `10`; the Agent protocol remains `1.1`.

Direct upgrade sources are Controller SQLite schemas 1 through 10 and earlier StackPilot 0.3 preview releases. This release has no database migration. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.37.sqlite3
```

Deploy Controller and Web together. Confirm the ACL role editor shows only granted permissions for built-in roles while custom roles remain editable. Verify proxy and security settings at desktop and mobile widths, including detail drawers, long values and a complete 10-second polling interval. Run `npm run audit:verify --workspace @stackpilot/controller` after cutover.

To roll back to `0.3.0-preview.35`, stop the new Controller, keep the verified schema 10 backup available, switch Controller and Web to the previous release, restore the previous systemd units if they changed, run `systemctl daemon-reload`, and restart. A database restore is not normally required because this release does not change schema.
