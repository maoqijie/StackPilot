# Upgrade to 0.3.0-preview.15

Status: preview. This release fixes abort-listener retention in the Agent database runtime. It does not change the Controller database schema, Agent protocol or helper request contracts.

Direct upgrade sources are Controller SQLite schemas 1 through 8 and earlier StackPilot 0.3 preview releases. Production deployment requires Node.js 22.x. Before changing the running release, verify the current SQLite database, create an online backup with its SHA-256 sidecar, and preserve Controller configuration, TLS material and the master key. Then run:

```bash
npm run release:preflight
npm run release:upgrade -- /protected/backups/pre-0.3.0-preview.15.sqlite3
```

Deploy Controller and Web first, then restart the database-helper and Agent after switching their versioned release links. Confirm the Helper process working directory points to the new release, the Agent remains non-root with the `systemd-journal` supplementary group, and heartbeats continue at the configured interval. Observe the Agent across at least two database collection intervals and confirm no new `MaxListenersExceededWarning` or database backup schedule synchronization warning appears.

Schema 8 remains unchanged. To roll back, switch Controller/Web, database-helper and Agent release symlinks to the previous compatible release and restart the affected services. Database restoration is required only if a separate schema-changing release was crossed during the same maintenance window.
